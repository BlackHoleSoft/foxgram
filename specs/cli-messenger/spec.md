# CLI Messenger (foxgram-cli)

CLI интерфейс для проверки работы бэкенда мессенджера и модуля шифрования foxgram-core

Корневая директория: foxgram-cli/

Для начала работы нужно вызвать:
`foxgram-cli`

Должен запускаться через npx.

## Идея

В мессенджере не предусмотрена авторизация и регистрация. Пользователь может отправить сообщение кому угодно, при условии, что он знает связку "подпись - публичный ключ" получателя.
Получатель хранит список переписок, если ему пишет пользователь не записанный в контактах, он не увидит сообщений.

Система хранит список контактов, куда можно добавить других пользователей.

Идентификатор пользователя - строка в формате `user_signature:user_pub_key`

## Интерфейс

### Первичная настройка

При запуске система проверяет наличие и содержимое файла `user.json`. Если его нет или структура не валидна (проверить через zod), то система просит ввести имя пользователя.

После этого система генерит:
- keypair
- подпись
Подпись вычисляется следующим образом:
```
const nameEncrypted = await core.createMessage({
    message: 'Alice', // user name
    senderPrivateKey: keypair.privateKey, // keypair - только что сгенерированная пара
    senderPublicKey: keypair.pubKey,
    receiverKey: keypair.pubKey
});
const signature = nameEncrypted.receiverPayload;
```

Теперь данные записываются в user.json
```
{
    "username": "alice",
    "userSign": "signature:pub_key",
    "publicKey": "pub_key",
    "privateKey": "private_key"
}
```

### Стартовый экран

Отображается при наличии и валидности `user.json`

После запуска пользователю отображается стартовый экран с выбором:
```
Foxgram CLI. Type a number to choose option
--- you can see or edit your user credentials in ./user.json file ---

1. Start messaging
2. Add contact
```

По умолчанию выбирается п.1
Пользователь должен ввести цифру и нажать Enter

Start messaging:
- пользователь выбирает пользователя из списка контактов (вводит номер пункта) и нажимает enter
- переходит в режим чата

Add contact:
- система предлагает ввести идентификатор пользователя
- система предлагает ввести имя пользователя (как его назвать в списке контактов)
- добавляет контакт в contacts.json
- возвращает пользователя на стартовый экран

contacts.json:
```
{
    "contacts": [
        {
            "name": "Bob",
            "id": "XXxkYNOTRQIJeNhWhQ2MpCrNzxVkySmArsfjeVV5dbcAuJtD3mq3npF6+aPs:6mgyS5uOvsmFojzdi8YYjEhKW6D+pJ5dvm6AMlF82Eg="
        }
    ]
}
```

### Режим чата

В режиме чата на экране появляется строка ввода сообщения:
`>> _`

После нажатия enter отправляется сообщение. При успешном статусе ответа сервера сразу выводим сообщение пользователя.
После каждого сообщения нужно делать отступ в одну строку.

Список сообщений периодически обновляется.

В момент отправки сообщения вместо строки ввода показываем loader (крутящиеся точки)

Каждое сообщение выводится в формате: `username::date:: message`
Для каждого пользователя необходимо предусмотреть свой цвет сообщения.
- для текущего пользователя зеленый
- для собеседника желтый

username берем из:
- для текущего пользователя из user.json
- для собеседника из списка контактов, если нет - из сообщения (parsedMessage.username)

date берем из сообщения, выводим в ISO формате в текущем часовом поясе, например: 2026-01-01T12:06:12

### Отправка сообщения

Перед отправкой получаем public_key получателя из его идентификатора:
```
const receiverPubKey = receiverId.split(':')[1];
```

Далее, шифруем сообщение и отправляем на сервер, указав senderId и receiverId (идентификатор - это пара `user_signature:user_pub_key`)

Формат сообщения:
```
const pocket = {
    sign: receiverId,
    message: typedMessage,
    sendDate: currentDate
};
```
sendDate - текущая дата формирования сообщения

> пример шифровки и расшифровки смотреть в testclient\index.js

### Получение сообщений

Для обновления списка сообщений нужно выполнять long polling с интервалом 10 секунд.
Запрашиваем отдельно сообщения от собеседника, и отдельно свои сообщения запросом POST /api/v1/messages/get
В момент первичной загрузки подгружаем до 100 сообщений.

Если сервер не отвечает или возникла ошибка, делаем 3 попытки с интервалом 3 секунды, после чего выводим ошибку.

Для расшифровки берем public_key отправителя из pocket.sign:
```
const receivedPubKey = pocket.sign.split(':')[1];
```

Если при расшифровке сообщения от собеседника возникла ошибка, игнорируем сообщение и не выводим.
Если при расшифровке своего сообщения возникла ошибка, выводим ошибку.
